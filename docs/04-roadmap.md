# 04 — Roadmap: фазы, критерии перехода, метрики

Принцип метода: **сначала канал → идея под него → потом код**, и не расширяться, пока текущая фаза не «выстрелила». Не строй следующую фазу раньше её критерия перехода.

## Фаза 0 — Подготовка (человек + агент, ~несколько дней)
Аккаунты и репозитории. Часть — на человеке (см. `03-monetization.md`).
- [ ] Chrome Web Store developer account (разовый взнос ~$5).
- [ ] Paddle vendor (verified), product + lifetime price.
- [ ] Supabase проект.
- [ ] Vercel проект под сайт-спутник.
- [ ] Репозиторий-монорепо или два репо (extension / site) + Supabase-папка.

**Критерий выхода:** аккаунты живы, секреты в `.env`, скелет проекта собирается.

## Фаза 1 — MVP (~2 месяца, ядро ценности)
Собрать и выпустить работающее расширение + минимальный сайт.
- [ ] Открытие PDF (drag&drop / picker / контекст-меню).
- [ ] Подпись (draw / type / upload), 2–3 сохранённые.
- [ ] Заполнение (текст, чекбоксы, даты, инициалы).
- [ ] Мини-правка страниц (удалить/повернуть/переупорядочить).
- [ ] Экспорт готового PDF (локально).
- [ ] Пейволл: счётчик бесплатных действий + гейт экспорта.
- [ ] Лицензия: Paddle checkout → Supabase → выдача → валидация (подход A, офлайн).
- [ ] Сайт-спутник: лендинг + privacy policy + checkout.
- [ ] Листинг стора: название, описания, 5 скриншотов (см. `pdf-sign-aso-listing.md`).
- [ ] Прогон на минимум разрешений и отсутствие удалённого кода (иначе отказ ревью).

**Критерий выхода:** расширение прошло ревью и опубликовано; покупка end-to-end работает и разблокирует офлайн.

## Фаза 2 — Трафик и ASO-итерация (после первых установок)
- [ ] «How to»-страницы на сайте под `how to sign a pdf`, `sign pdf without adobe` и т.п.
- [ ] Итерация листинга: A/B названия/иконки/скриншотов, работа с первыми отзывами.
- [ ] Мягкие in-app подсказки оставить отзыв (стор ранжирует по инсталлам/рейтингу).
- [ ] Отдельные лендинги под конкретные документы (lease/W-9/contract) — по мере данных.

**Критерий входа:** листинг живой, идут органические установки. **Не** начинать раньше публикации.

## Фаза 3 — Расширение охвата (после устойчивого органического потока)
- [ ] Больше гео/языков (UK/DE/FR — там мы ещё не снимали частотности, но спрос есть).
- [ ] Возможная веб-версия «magnet» на сайте (часть функций бесплатно в браузере как приманка).
- [ ] Разбор новых keyword-кластеров через тот же процесс валидации.

**Критерий входа:** органика стабильна и профитна.

## Правило про платную рекламу
Платные каналы **не трогаем**, пока органика не выйдет в плюс. Это прямое правило метода: сначала бесплатный трафик до профитности, потом уже эксперименты с рекламой.

## Метрики (что трекать с Фазы 1)
| Метрика | Зачем |
|---|---|
| Установки / неделя | рост, топливо для ранга в сторе |
| Impressions → install CTR в сторе | качество листинга (название/иконка/скрин) |
| Активация: % дошедших до первого успешного экспорта | продукт «доносит» ценность? |
| Free → paid конверсия | работает ли пейволл/цена |
| Refund rate | не слишком ли агрессивен пейволл / ожидания |
| Рейтинг и кол-во отзывов | ранжирование + доверие |

Всё измерять приватно и легко (без тяжёлого трекинга — это противоречит клину).

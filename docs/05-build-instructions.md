# 05 — Инструкции агенту: как строить

Читается **после** `README.md`, `01`–`04`. Это твой план исполнения. Строй итеративно по milestones, каждый с критерием готовности (DoD). Не перескакивай вперёд.

## Общие правила
- **Стек фиксирован** (см. `02`): MV3 + TypeScript + Vite, `pdf.js` + `pdf-lib` **в бандле**. Не тяни код с CDN.
- **Клин — закон:** файл пользователя не покидает устройство. Любая фича, требующая отправки файла на сервер, — стоп.
- **Минимум разрешений** в манифесте. Добавляешь разрешение — обоснуй в комментарии, зачем.
- **Секреты — в `.env`**, никогда в коде и не в бандле расширения. Сделай `.env.example` с плейсхолдерами.
- **Ты не создаёшь аккаунты и не вводишь платёжные/личные данные.** Где это нужно (Paddle, Supabase, Chrome dev account, реальные ключи) — ставь заглушку и **явно проси человека** выполнить шаг. Список таких шагов веди в `HUMAN-TODO.md`.
- Пиши типы, мелкие чистые модули, комментарии на английском в коде.

## Milestone 1 — Скелет и рендер PDF
- Инициализируй проект: Vite + TS + MV3-манифест (каркас из `02`).
- Editor page рендерит многостраничный PDF через `pdf.js` на canvas (drag&drop + файл-пикер).
- **DoD:** любой локальный PDF открывается и листается; в Network нет исходящих запросов с содержимым файла.

## Milestone 2 — Оверлей и подпись
- Оверлей-слой поверх страниц: размещение/перетаскивание/масштаб элементов.
- Подпись тремя способами (draw / type / upload) → PNG; 2–3 сохранённые в `chrome.storage.local`.
- **DoD:** подпись можно поставить, подвигать, удалить; сохранённые подписи переживают перезапуск.

## Milestone 3 — Заполнение и правка страниц
- Текстовые поля, чекбоксы/крестики, даты, инициалы на оверлее.
- Мини-правка: удалить/повернуть/переупорядочить страницы.
- **DoD:** на многостраничном бланке проставляются поля; правки страниц отражаются в предпросмотре.

## Milestone 4 — Экспорт
- `pdf-lib`: взять исходные байты + оверлеи → embed подписи, drawText/дат/чекбоксов, применить правки страниц → новые байты.
- Скачивание через `chrome.downloads` или `<a download>`.
- **DoD:** скачанный PDF содержит все элементы в правильных местах; открывается в сторонних просмотрщиках; сеть не задействована.

## Milestone 5 — Пейволл
- Счётчик бесплатных действий (4–5) в `chrome.storage.local`; гейт экспорта после лимита.
- Экран пейволла с кнопкой на checkout (сайт-спутник / Paddle).
- **DoD:** после лимита экспорт заблокирован; клик ведёт на checkout; счётчик не сбрасывается тривиально (напр. простым релоадом).

## Milestone 6 — Лицензия (Supabase + Paddle, подход A)
- Supabase: таблица `licenses`, Edge Function приёма вебхука Paddle с **верификацией подписи вебхука**, выпуск подписанного токена (Ed25519).
- Расширение: Options-страница для ввода ключа/токена; **локальная проверка подписи** публичным ключом из бандла; кэш; периодическая онлайн-перепроверка статуса (best-effort, офлайн не наказываем).
- **DoD:** тестовая покупка (Paddle sandbox) → выдача → ввод в расширении → пейволл снят; разблокировка сохраняется **офлайн**.
- **Human-TODO:** реальные Paddle/Supabase аккаунты, ключи, webhook URL/секрет, пара ключей подписи.

## Milestone 7 — Сайт-спутник (Next.js/Vercel)
- Лендинг (клин, скриншоты, «Add to Chrome» + «Buy lifetime»), **privacy policy**, success-страница с ключом.
- Интеграция Paddle checkout.
- **DoD:** сайт задеплоен на Vercel; privacy policy доступна по URL (нужна для листинга стора); checkout ведёт к выдаче лицензии.

## Milestone 8 — Подготовка к публикации
- Аудит: минимум разрешений, нет строк `http(s)://` с исполняемым кодом, CSP корректный, privacy policy указана.
- Иконки, 5 скриншотов, тексты листинга (из `pdf-sign-aso-listing.md`).
- **DoD:** сборка проходит self-check по чек-листу из `pdf-sign-aso-listing.md` §7; пакет готов к загрузке (загрузку в стор делает человек).

## Проверки перед каждым коммитом
- [ ] Нет удалённого кода (грепни собранный бандл на `http://` / `https://` в script-контексте).
- [ ] Нет исходящих сетевых запросов с содержимым PDF.
- [ ] Не добавлены лишние разрешения.
- [ ] Секреты не попали в бандл/репозиторий.

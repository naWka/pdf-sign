# 03 — Монетизация: пейволл, Paddle, Supabase

## Модель
- **Free → lifetime unlock.** Первые **4–5 «сохранений/подписей»** бесплатны (считаем в `chrome.storage.local`). Дальше экспорт готового PDF гейтится за разовую оплату.
- **Разовый платёж, доступ навсегда.** Никаких подписок. Цену определяет владелец продукта (не хардкодить в доках — брать из Paddle price ID). Это одноразовая задача, поэтому lifetime уместнее подписки.
- Бесплатный тариф специально щедрый на *установки и отзывы* (стор — это поисковик, инсталлы двигают ранг), но узкий на *результат* (экспорт).

## Роли компонентов
- **Paddle** — Merchant of Record: checkout в браузере, приём платежа, **расчёт и уплата налогов/VAT в US/EU**, инвойсы. Мы не касаемся налоговой рутины.
- **Supabase** — Postgres + Edge Functions: приём вебхука Paddle, генерация/хранение лицензии, эндпоинт валидации.
- **Расширение** — считает бесплатные действия, гейтит экспорт, принимает ключ, проверяет лицензию (в идеале офлайн, локально).

## Поток покупки (end-to-end)
```
1. Юзер упёрся в пейволл в расширении → клик «Unlock lifetime»
        │  (открываем сайт-спутник / Paddle checkout в новой вкладке)
        ▼
2. Paddle Checkout в браузере (юзер вводит оплату у Paddle, НЕ у нас)
        │
        ▼
3. Paddle шлёт webhook → Supabase Edge Function
        │  Edge Function ВЕРИФИЦИРУЕТ подпись вебхука Paddle
        ▼
4. Edge Function создаёт лицензию:
   - пишет запись в таблицу licenses (Postgres)
   - генерирует лицензионный КЛЮЧ (напр. UUID) и/или
   - выпускает ПОДПИСАННЫЙ токен (Ed25519/RS256) с payload {license_id, issued_at}
        │
        ▼
5. Ключ/токен показывается на success-странице и/или шлётся на email (через Paddle или Supabase)
        │
        ▼
6. Юзер вставляет ключ в Options-странице расширения
        │
        ▼
7. Расширение валидирует (см. ниже) → снимает пейволл, кэширует статус
```
Важно: **платёжные данные вводит пользователь у Paddle в браузере.** Ни расширение, ни агент их не касаются.

## Валидация лицензии — два подхода

### A. Подписанный токен (рекомендуется — работает офлайн) ✅
Соответствует нашему обещанию «works offline».
- Supabase Edge Function подписывает компактный payload приватным ключом (Ed25519).
- В расширение **бандлится ПУБЛИЧНЫЙ ключ**.
- Расширение проверяет подпись токена **локально**, без сети → разблокировка работает офлайн.
- Кэш в `chrome.storage.local`.
- **Периодическая онлайн-перепроверка** (напр. раз в 7–14 дней, best-effort): дёрнуть публичный эндпоинт статуса, чтобы уважать возвраты/отзыв. Если сети нет — доступ остаётся (не наказываем офлайн-юзера).

### B. Публичный эндпоинт валидации (проще, но требует сети)
- Юзер вставляет ключ → расширение зовёт **публичный** Supabase-эндпоинт валидации (без секретных ключей во фронте).
- Ответ кэшируется в `chrome.storage.local` с TTL; периодически перевалидируется.
- Минус: первичная разблокировка требует онлайна.

**Выбор по умолчанию: A** (офлайн-дружелюбно, лучше ложится на клин). B — если нужно быстрее выкатить, с планом миграции на A.

## Данные в Supabase (ориентир схемы)
```sql
create table licenses (
  id            uuid primary key default gen_random_uuid(),
  paddle_txn_id text unique not null,     -- идемпотентность по вебхуку
  email         text,
  status        text not null default 'active', -- active | refunded | revoked
  created_at    timestamptz default now()
);
```
- Уникальность по `paddle_txn_id` → защита от дублей вебхука.
- Валидационный эндпоинт отдаёт только `status`, без чувствительных полей.
- Секреты (Paddle API key, приватный ключ подписи) — только в Edge Functions / Supabase Vault, **никогда** во фронте расширения.

## Anti-piracy — прагматично
Это дешёвый локальный инструмент, не банк. Не переусердствуй:
- Подписанный токен + периодическая проверка отсекает 95% casual-обхода — достаточно.
- Не строй тяжёлую device-fingerprint-систему в MVP.
- Возвраты/отзыв ловим через статус лицензии и перепроверку.

## Что понадобится от человека (агент НЕ создаёт сам)
- Аккаунт **Paddle** (verified vendor), product + price ID (lifetime).
- Проект **Supabase** (URL, anon key, service role key — server-only).
- Пара ключей подписи (сгенерировать; приватный — в Supabase Vault, публичный — в бандл).
- Настроенный **webhook** Paddle → Supabase Edge Function (URL и секрет вебхука).
Агент готовит код и `.env.example`, оставляет плейсхолдеры и **явно просит человека** подставить секреты и завести аккаунты.

# Sign & Fill PDF — пакет документации для сборки MVP

Этот пакет — полный бриф для кодового агента. Прочитай **все файлы по порядку перед началом работы**: они описывают *что* мы делаем, *зачем*, *как* и *в каком порядке строить*.

## Что это за продукт (в одном абзаце)
Chrome-расширение, которое даёт подписать и заполнить любой PDF **прямо в браузере, локально**: файл никогда не покидает устройство, регистрация не нужна. Монетизация — один платёж навсегда (lifetime), а не подписка. Рынок — US/EU. Спрос подтверждён (десятки тыс. запросов/мес по кластеру «sign pdf», растёт).

## Технологический стек (кратко)
- **Расширение:** Manifest V3, TypeScript, сборка через Vite. UI на vanilla TS или лёгком React (без фреймворка в service worker).
- **PDF:** `pdf.js` (рендер) + `pdf-lib` (правка/экспорт) — **бандлятся внутрь**, не с CDN (MV3 запрещает удалённый код).
- **Монетизация:** Paddle (Merchant of Record, берёт налоги/VAT) + Supabase (Edge Functions + Postgres) для выдачи и валидации лицензий.
- **Сайт-спутник:** Next.js на Vercel — лендинг, checkout, privacy policy, «how to»-страницы.

## Железные правила (не нарушать)
1. **Файл пользователя никогда не уходит на сервер.** Вся работа с PDF — клиентская. Это и есть продукт.
2. **Никакого удалённого кода** (MV3): все библиотеки в бандле.
3. **Минимум разрешений** в манифесте — каждое лишнее бьёт по конверсии и по ревью.
4. **Без аккаунтов внутри продукта** — лицензия по ключу, а не по логину.
5. **Агент не создаёт аккаунты и не вводит платёжные данные** — эти шаги помечай и отдавай человеку (см. `05-build-instructions.md`).

## Глоссарий
- **Клин (wedge)** — единственное отличие, вокруг которого строится всё: *no upload, no account, local & private.*
- **MoR (Merchant of Record)** — Paddle как продавец записи: берёт на себя расчёт и уплату налогов в US/EU.
- **Lifetime** — разовая оплата, доступ навсегда (для одноразовой задачи это уместнее подписки).
- **Сайт-спутник** — маркетинговый сайт рядом со стором: ловит Google-трафик и держит checkout.
